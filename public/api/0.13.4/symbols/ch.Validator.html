ch.Validator (Class)

Documentation updated on Tue Jul 30 2013, generated by JsDoc Toolkit.

Overview

ch.Validator extends ch.Widget. Validator is an engine for HTML forms elements.

Requires

  • ch.Condition

Returns

itself

Configuration

The following options lets you create a customized ch.Validator:

  • conf: Object Object with configuration properties.
  • conf.conditions: Object Object with conditions.

Returns

Properties

conditions

This public Map saves all the validation configurations from this instance.

object

element

Reference to a DOM Element. This binding between the component and the HTMLElement, defines context where the component will be executed. Also is usual that this element triggers the component default behavior.

HTMLElement

type

This public property defines the component type. All instances are saved into a 'map', grouped by its type. You can reach for any or all of the components from a specific type with 'ch.instances'.

String

uid

The 'uid' is the Chico's unique instance identifier. Every instance has a different 'uid' property. You can see its value by reading the 'uid' property on any public instance.

number

Events

afterValidate
Triggers when the validation process ends.

Examples

widget.on("afterValidate",function(){
	submitButton.disable();
});
beforeValidate
Triggers before start validation process.

Examples

widget.on("beforeValidate",function(){
	submitButton.disable();
});
error
Triggers when an error occurs on the validation process.

Examples

widget.on("error",function(event, condition){
	errorModal.show();
});
ready
Triggers when the component is ready to use.

Examples

Following the first example, using widget as modal's instance controller:

widget.on("ready",function(){
	this.show();
});
